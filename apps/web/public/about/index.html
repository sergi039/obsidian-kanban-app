The Reconciler

On startup, the reconciler parses each board's .md file and syncs tasks into a local SQLite database. Each task is identified by a stable <!-- kb:id=xxxxxxxx --> marker injected into the markdown, ensuring tasks survive reordering and editing in Obsidian.

File Watcher

A filesystem watcher monitors your vault files. When you edit a task in Obsidian, the watcher detects the change, re-runs the reconciler for that file, and pushes updates to all connected browsers via WebSocket â€” in real-time.

Write-Back

When you make changes in the web UI (move a card to Done, change priority, etc.), those changes are written back to the .md file atomically. Priority uses emoji markers (ğŸ”º = urgent, â« = high), and Done status toggles the checkbox ([ ] â†” [x]).
